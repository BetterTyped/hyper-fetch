## Introduction

Thanks to this hook, we are able to manipulate the [`Cache`](/docs/Docs/Managers) of the passed command and receive
change callbacks. It is important to make sure that `cacheKey` is not affected by auto-generation - to make sure we have
the stable connection between hook's command and the cache.

```tsx
const { data, error, loading, revalidate, onSuccess, onError, onFinished } = useCache(getUseres);

revalidate();

onSuccess((payload) => {
  console.log(payload); // [ User, User, User ]
});

onError((error) => {
  console.log(error); // { message: string }
});

onFinished(([payload, error, status]) => {
  console.log(payload); // [ User, User, User ] | null
  console.log(error); // { message: string } | null
  console.log(status); // 200 / 400 / 404 / 500 ...
});
```
